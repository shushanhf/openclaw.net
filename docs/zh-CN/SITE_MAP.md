## 建议的落地路径

1. 从 [README.md](../README.md) 开始获取产品级概览。
2. 当评估者需要最短的技术导向时，将他们引导到 [START_HERE.md](START_HERE.md)。
3. 将实践用户引导到 [QUICKSTART.md](QUICKSTART.md)，如果需要仓库上下文，则引导到 [GETTING_STARTED.md](GETTING_STARTED.md)。
4. 在任何公共部署之前，将运维人员链接到 [SECURITY.md](../SECURITY.md)、[COMPATIBILITY.md](COMPATIBILITY.md) 和 [RELEASES.md](RELEASES.md)。
5. 将贡献者链接到 [CONTRIBUTING.md](../CONTRIBUTING.md)、[ROADMAP.md](ROADMAP.md) 和 [ai-contributor-guide.md](ai-contributor-guide.md)。

## 构建说明

- 将页面导入静态站点生成器时保留现有的相对链接。
- 将 `docs/experiments/` 下的文件视为历史研究笔记，而非主导航。
- 将 [docs/README.md](README.md) 保留为 GitHub 浏览的源索引页面。
- 如果网站框架规范化 slug，为小写路由名称添加重定向或别名。
- 在源代码控制中将安全和贡献页面保留在仓库根目录中，即使网站将它们呈现在运维或项目分区下。
