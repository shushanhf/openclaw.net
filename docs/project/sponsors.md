# Sponsors

OpenClaw.NET welcomes sponsorship from individuals, companies, and organizations that want to support a practical, self-hostable, vendor-neutral .NET agent runtime and gateway.

## What Sponsorship Supports

Sponsorship can support:

- development
- documentation
- examples
- infrastructure
- release work
- community support
- testing and compatibility work

Sponsors help sustain the project, but sponsorship does not change project governance or technical authority.

## What Sponsorship Does Not Provide

Sponsorship does not provide:

- roadmap control
- release authority
- maintainer status
- exclusive rights
- ownership of an ecosystem direction
- vendor-specific defaults
- priority authority over architecture or security decisions

Maintainer status is based on sustained contribution, trust, technical judgment, and community conduct, not sponsorship.

## Sponsorship and Commercial Use

Downstream commercial products are welcome.

Commercial contributors and sponsors may build products, integrations, internal platforms, adapters, services, or deployment offerings on top of OpenClaw.NET. The upstream project must remain vendor-neutral, self-hostable, and reusable.

When a contribution directly supports a company or customer use case, contributors should disclose that context in the related issue, pull request, or proposal so maintainers can evaluate scope, neutrality, security posture, and whether the work belongs in core or an extension.
